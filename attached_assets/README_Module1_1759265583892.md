# Module 1 — API Contracts & Domain Skeleton (No external deps)

**Purpose:** Extend Module 0 by adding *foundational MVP domain endpoints* with strict validation and a unified error taxonomy.
We still avoid external integrations. Data is stored in-memory. This keeps the dev loop fast and safe on Replit.

## What’s included
- **Unified errors** (`/server/errors.js`) with typed API error payloads.
- **In-memory db** (`/server/db.js`) for `profiles` and `interviews`.
- **Routes**
  - `POST /api/v1/profiles` — create profile (name, email).
  - `GET /api/v1/profiles/:id`
  - `GET /api/v1/profiles` — list with `?q` (search), `?limit`, `?cursor` (simple cursor).
  - `PATCH /api/v1/profiles/:id` — partial update.
  - `POST /api/v1/interviews` — create interview (profileId, title).
  - `GET /api/v1/interviews?profileId=...` — list by profile.
- **Validation** with Zod; consistent 4xx/5xx responses.
- **Self-tests** extended to cover profile creation, list, and interview creation.

## Exit Criteria
- ✅ Health still returns ok.
- ✅ Creating a profile returns 201 with normalized payload.
- ✅ Listing profiles returns created profile.
- ✅ Creating an interview linked to the profile returns 201.
- ✅ `npm test` passes; `/logs/selftest.json` shows all green.
