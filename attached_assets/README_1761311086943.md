# Profile Editor Enhanced — Replit Drop-in

This update merges **all sections** from your design into `profile_edit_enhanced.html`,
keeps local-only assets, replaces Material Symbols with inline SVGs, and renames
the availability header to **Set Your Availability**.

## Files
- `profile_edit_enhanced.html` — Drop this into your public/static folder to replace the current page.

## Why this fixes CI/headless issues
- No CDNs (no Google Fonts / Material Symbols / Tailwind CDN).
- Uses your local `/css/tailwind.css` and `/js/safe-render-shim.js`.
- Inline SVG icons (chevron, upload, video, trash, file, plus).

## Save & Return wiring
The page still dispatches `profile:save-request` and then redirects back to the template page:
`/profile_v4_1_package/public/index.html?id=...`.

## Deploy
1. Copy `profile_edit_enhanced.html` into your app's static root where the existing page lives.
2. Ensure these local assets are available:
   - `/css/tailwind.css`
   - `/js/safe-render-shim.js`
   - `/js/profile_edit.bind.js` (optional; missing file won't block render)
3. Open the page locally to verify:
   - `http://localhost:5000/profile_edit_enhanced.html?id=dev_mock`
