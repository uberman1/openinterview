# Guardrails for Deploying `test2.html` in Replit

**Goals**
- Add a QA index page without touching protected templates (e.g., `home.html`) and without breaking guardrails.
- Keep total files well under Replit’s 20-file limit.

## Rules
1) **Do not edit protected files.** No direct changes to `home.html`, header includes, or other guardrailed templates.
2) **Static placement only.** Place `test2.html` in the public root so it is reachable at `/test2.html`.
3) **No secret exposure.** This page is read-only and must not display API keys, tokens, or internal IDs.
4) **No indexing.** `test2.html` already includes `<meta name="robots" content="noindex, nofollow">`.
5) **Admin-only entry point.** If you need a link from the app, use *response-time JavaScript injection* on `/#/admin` (non-persistent) — not code edits.

## Admin Entry (Non-Persistent Injection)
Paste the following in your browser console while on `/#/admin`:
```js
(() => { 
  const id = "qa-index-link";
  if (document.getElementById(id)) return;
  const btn = Object.assign(document.createElement("a"), {
    id,
    href: "/test2.html",
    target: "_blank",
    textContent: "QA Index",
    style: "margin-left:10px;padding:6px 10px;border:1px solid #111;color:#111;text-decoration:none;border-radius:6px;font-size:12px;"
  });
  const header = document.querySelector("header, .admin-actions, nav") || document.body;
  header.appendChild(btn);
})();
```
This avoids modifying protected files and keeps QA access for admins.

## Validation
- `/test2.html` loads with white background / black text and the seven sections.
- No protected files changed; app continues to pass guardrails checks.
- File count remains minimal.
