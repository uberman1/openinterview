# OI-UI-Scroll-Hero-v1 — Guardrail Update (Self‑Deploying)

This package applies the scrolling fix, preserves Resume + Highlights visibility, and adds smooth **75% hero shrink on scroll** — with **guardrails** so auto‑tools only touch the marked regions.

## What it does
1. Keeps **Resume** + **Highlights** visible and unchanged.
2. Only the content **below the Resume** scrolls (desktop by default).
3. The hero video smoothly shrinks to **75%** as the user begins scrolling, and restores at the top.
4. All inserted/changed code is wrapped with **OI-UI-Scroll-Hero-v1** start/end markers.

## Quick Start
```bash
unzip oi-ui-scroll-hero-v1.zip -d oi-update
cd oi-update
bash deploy.sh /absolute/path/to/profile.html
```

- A backup will be written next to your file: `profile.html.bak`.
- The script is **idempotent**; re-running won’t duplicate markers.

## Files
- `deploy.sh` – convenience runner
- `scripts/apply_update.py` – safe patcher
- `snippets/css.txt` – CSS payload
- `snippets/js.txt` – JS payload
- `selftest.json` – simple verification spec

## Rollback
Restore the backup:
```bash
cp /absolute/path/to/profile.html.bak /absolute/path/to/profile.html
```
or remove the guarded blocks between:
```html
<!-- OI-UI-Scroll-Hero-v1:START -->
<!-- OI-UI-Scroll-Hero-v1:END -->
```

## Notes
- The patcher looks for your hero wrapper: a `<div>` containing the classes `relative aspect-video w-full`.
- It will add `id="heroWrapper"` and the sticky classes.
- It will add `id="heroInner"` to the inner overlay div matching `absolute inset-0 bg-black/30 …`.
- It finds the Resume section by locating the `<h2>Resume</h2>` header and inserts a **below-resume** scroll container **after** that section’s closing `</section>`.
- CSS is injected into your first `<style>` block if present, otherwise a new one is created in `<head>`.
- JS is injected before `</body>`.
