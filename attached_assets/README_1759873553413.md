# Header Unifier v5 — Make every page match the Home header

**Goal**: Same spacing, positioning, avatar and navigation on **Home, Availability, Profiles, Uploads, Subscription, Password**.

### What it does
- Replaces the first `<header>` on all pages **except** Availability with a **canonical Home-style header**:
  - `container mx-auto px-10 py-4 flex items-center justify-between whitespace-nowrap`
  - Logo SVG + brand text (left)
  - Center nav: Home • Availability • Profiles • Uploads • Subscription • Password • Log Out
  - 40px round avatar (right)
- **Availability** keeps its page title “Availability” and Revert/Save. The topbar is **inserted above** that page header.
- Removes stray CTAs (“New Profile”, “New Interview”, “Sign In”).
- No template files changed — binder is injected at response-time (guardrail-safe).

### Install (Replit)
```bash
unzip header_unify_v5.zip -d /tmp/huv5 && cp -R /tmp/huv5/* .
node scripts/apply_header_unify_v5.mjs
# restart your server (e.g., npm run dev)
npm i supertest cheerio
node scripts/selftest_header_unify_v5.mjs
```

### Notes
- The avatar uses any existing background-image it finds; otherwise it falls back to your default image.
- If you previously had duplicate menus, this replaces the whole header with one canonical instance (except on Availability, where it prepends the topbar and leaves your page header intact).
