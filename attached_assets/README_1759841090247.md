# Home-as-Profile Update — Patch

**What this does**
1) Makes `home.html` the main landing page by aliasing **/profile(.html)** → **/home.html**.
2) Adds visible **Edit / View / Delete** actions to every row in **My Interviews** and **My Resumes** (injected at runtime without changing your HTML).
3) Adds your **Upcoming Interviews** table and loads real data from the first available endpoint among:
   - `/api/interviews/upcoming`
   - `/api/bookings/upcoming`
   - `/api/schedule/upcoming`
4) Keeps the numeric stats; removes sparkline/bar/pie charts per your instruction.

**Apply (Replit)**
```bash
unzip home_as_profile_update_patch.zip -d /tmp/oi_home && cp -R /tmp/oi_home/* .
node scripts/apply_home_as_profile_update.mjs
# restart (e.g., npm run dev)
npm i supertest cheerio
node scripts/selftest_home_as_profile.mjs
```

**Rollback**
```bash
cp -f public/home_prev.html public/home.html
# Optionally remove the HOME_BIND_INJECT block from index.js.
```

**Actions Wiring (progressive fallback)**
- Interviews:
  - **View** → `/interviews/{id}` → `/interview/{id}` → `/interviews/view?title={title}` → `/profile_share.html` → `/profile_public.html`
  - **Edit** → `/interviews/{id}/edit` → `/profile_edit.html?interview={title}` → `/profile_edit.html`
  - **Delete** → `DELETE /api/interviews/{id}` → `DELETE /api/interviews?title={title}`
- Resumes:
  - **View** → `/uploads/{filename}` → `/files/{id}` → `/api/files/{id}/download`
  - **Edit** → `/uploads.html` or `/uploads` (adds `#file=name` anchor)
  - **Delete** → `DELETE /api/files/{id}` → `DELETE /api/uploads?name={filename}`
